# MVP Scope — Qoldau AI

> Версия документа: v0.3.0 — Demo MVP
> Назначение: зафиксировать границы Demo MVP, чтобы продуктовая глубина не страдала из-за технических ограничений.

## Что такое Demo MVP

Demo MVP — это **полноценный frontend-продукт на mock-данных**, который показывает будущую систему целиком, но без серверной части.

Цель: открыть ссылку и провести полный сценарий Родитель → Ребёнок → Тьютор → Специалист → Отчёт → Event Timeline без тупиков и без объяснений «тут пока ничего нет».

## Что входит в Demo MVP

### Продуктовая глубина (полная)

- Все 4 роли: **Parent, Child, Tutor, Specialist**.
- Все основные экраны для каждой роли (см. `FEATURE_MAP.md`).
- 60–100 событий за 7 дней для основного demo-ребёнка (Алихан).
- 3 ребёнка в системе: Алихан, Мира, Тимур.
- Все типы событий покрыты: voice, food, water, toilet, sleep, behavior, sensory, communication, aac_card, phrase, media_request, sos, calm_mode, tutor_note, specialist_note, state.
- Все дневники читают из Event Timeline.
- Все действия ребёнка создают Event.
- Все AI-выводы формулируются осторожно.
- Все тупики закрыты (toast или navigate).
- Guided Demo Mode (18 шагов) проходит целиком.

## Что НЕ входит в Demo MVP

Список ограничений — они есть **по техническим причинам**, а не потому что продукт неполный.

- ❌ **Real backend / API** — все данные в Zustand store + localStorage.
- ❌ **Real auth** — переключение ролей через UI без логина.
- ❌ **Real STT** — голосовой ввод имитируется через `setTimeout` (3 секунды «записи» → фиксированный transcript).
- ❌ **Real LLM** — AI-парсер возвращает предзаготовленные события (`src/lib/aiParser.mock.ts`).
- ❌ **Real push-уведомления** — обратная связь через in-app toast.
- ❌ **Real cloud storage** — данные только в браузере.
- ❌ **Payment** — нет страниц оплаты.
- ❌ **Medical records** — никакой медицинской документации, диагнозов, лечения.
- ❌ **Wearable / GPS** — нет физических устройств.

## Демо-данные

В Demo MVP заложены:

- 3 ребёнка с разными профилями (Алихан 7 лет — основной, Мира 5 лет, Тимур 9 лет).
- 2 родителя, 2 тьютора, 2 специалиста.
- 60+ связанных событий за 7 дней.
- AI insights, сгенерированные по формулам в `demoDataset.ts`.

Эти данные видны при первом открытии приложения — пользователь может сразу показывать сценарий без подготовки.

## Что можно показывать

Demo MVP годится для:

1. **Демо инвесторам** — показать продуктовую глубину и UX.
2. **Семьям с детьми с РАС** — собрать обратную связь об интерфейсе.
3. **Коррекционным центрам** — показать, как будет выглядеть их рабочий процесс.
4. **Специалистам** — обсудить, как их работа отражается в продукте.

## Что НЕльзя показывать как «готовый продукт»

- ❌ «Это приложение, которое уже работает» — это mockup.
- ❌ «Здесь есть диагностика» — нет, и не будет. Только наблюдения.
- ❌ «Данные хранятся в облаке» — нет, в браузере.
- ❌ «Ребёнок может пользоваться сам без взрослого» — нет, всегда нужен взрослый-подтверждающий.

## Сценарии, которые работают целиком

См. `MVP_WALKTHROUGH.md` для полного скрипта показа.

Краткая сводка:

- **Parent flow**: VoiceObservation → AIReview → ClarifyingQuestions → confirmed Event в Timeline.
- **Child flow**: Карточка / фраза / SOS / Calm Mode → Event с in-app feedback.
- **Tutor flow**: VoiceObservation → AIReview → tutor_note Event → Report с copy/send.
- **Specialist flow**: фильтры по детям, ABC, Communication Profile, Support Plan, Reports.

## Сценарии, которые запланированы, но не реализованы в Demo MVP

- ❌ Регистрация / логин (auth).
- ❌ Подключение нескольких семей / организаций (multi-tenant).
- ❌ Календарь / расписание с пуш-уведомлениями.
- ❌ Видео/аудио-чат между родителем и тьютором.
- ❌ Экспорт отчётов в PDF / Excel.
- ❌ Интеграция с wearable-устройствами.
- ❌ Геозоны.
- ❌ Биллинг.

Все эти сценарии описаны в `PHASE_2_ROADMAP.md` и `PILOT_SCOPE.md`.
