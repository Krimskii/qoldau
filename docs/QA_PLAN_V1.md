# QA Plan — v1.0 Pilot

Сквозной ручной QA перед раздачей ~20 семьям. Проходить на реальном Android-устройстве
с прод-прокси (`VITE_API_BASE_URL=<prod>`), плюс в браузере для быстрых прогонов.

## 0. Предусловия
- [ ] Backend-прокси задеплоен, `/api/health`, `/api/ai/health` (`claude`),
      `/api/stt/health` (`whisper`) отвечают.
- [ ] APK собран с прод `VITE_API_BASE_URL`, подписан release-keystore, ставится.
- [ ] Устройство и (для LAN-теста) ПК в одной сети; либо прод-URL по HTTPS.

## 1. Core flow — голосовое наблюдение (главный сценарий)
- [ ] Parent → VoiceObservation → запись 5+ сек → stop.
- [ ] Состояние «обработка» показывается.
- [ ] Приходит транскрипт + структурированные события.
- [ ] Событие появляется в Event Timeline **один раз** (нет дублей).
- [ ] AI-формулировки осторожные («Похоже…», «не диагноз»).
- [ ] Повторная запись создаёт новое событие, старое не дублируется.

## 2. Fallback / оффлайн
- [ ] Backend недоступен (выключить/самолётный режим) → запись уходит в Web Speech /
      demo-fallback, приложение **не падает**.
- [ ] Микрофон запрещён → понятное поведение, без «страшной» технической ошибки.
- [ ] Полный оффлайн: приложение открывается, локальные данные видны.

## 3. Роли (переключение на одном устройстве)
- [ ] **Parent**: home, events, care diary, analytics, profile — открываются, RU-текст, без заглушек.
- [ ] **Child**: home, cards, speak, calm, need-карточки — крупные тач-таргеты, спокойная эстетика, без тревожных элементов.
- [ ] **Tutor**: home, voice, report — работают.
- [ ] **Specialist**: dashboard, ABC, communication profile, reports — работают.
- [ ] Выход из роли ребёнка работает (оба пути).

## 4. Онбординг / семья
- [ ] Первый запуск ведёт на настройку семьи (не на демо-лендинг).
- [ ] Ввод имени ребёнка → имя подставляется во всём приложении.
- [ ] Изменение имени в профиле работает.
- [ ] Consent-gate показан до первой записи; после согласия не повторяется.

## 5. Приватность (см. PRIVACY_CONSENT_PILOT.md)
- [ ] Мини-политика доступна из приложения.
- [ ] «Очистить данные» стирает локальные данные ребёнка.
- [ ] Нет облачного хранения/аналитики по ребёнку.

## 6. Дизайн / доступность
- [ ] Padding консистентен (`px-5`), тач-таргеты ≥44px (детский UI крупнее).
- [ ] Цвета из токенов, тёмная тема не протекает в детскую роль.
- [ ] Loading / empty / error состояния на Timeline, Voice, ролях.
- [ ] Статус-бар/safe-area корректны на Android (контент не под вырезом).

## 7. Мобильная упаковка
- [ ] APK ставится «из файла» / через Internal Testing.
- [ ] Аппаратная кнопка «назад» = history.back, не выкидывает из приложения.
- [ ] Разрешение микрофона запрашивается корректно.
- [ ] Кнопки запуска/иконки/splash отображаются.

## 8. Безопасность формулировок
- [ ] `grep` по продуктовому коду: нет запрещённых medical claims
      (лечит/диагностирует/исправляет/нормализует/манипулирует/неадекватное/
      поведенческое нарушение/«ИИ точно понял»). Disclaimer «не диагностирует» — есть.

## Выходной критерий v1.0
Все разделы 1–8 зелёные на реальном устройстве → Draft PR в master → Ready →
раздача пилотной группе.
