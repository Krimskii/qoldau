# Mock Data Specification — Qoldau AI

> Полная спецификация mock-данных, заложенных в Demo MVP.
> Источник истины: `src/data/demoDataset.ts`.

## Обзор

Demo MVP загружает 60+ связанных событий при первом открытии. Пользователь видит полноценный продукт без необходимости генерировать данные вручную.

Все данные — `demoDataset.ts`, который сидит в `useEventStore` через `seedDemoEvents()` при init.

| Тип | Кол-во | Описание |

| `water` | 4 | Запросы воды |

| `behavior` | 3 | Нервозность / переход |
| `communication` | 5 | Звуки «ма», «ту-ту», «ва» |

| `tutor_note` | 3 | Заметки тьютора |
| `specialist_note` | 1 | Заметка специалиста |
| `state` | 5 | Состояние (спокойный, проснулся) |

### Распределение по источнику (sourceRole)

| Источник | Кол-во |
|----------|--------|
| `parent` | 35 |
| `child` | 18 |
| `tutor` | 4 |
| `specialist` | 1 |
| `ai` | 0 (AI не создаёт события, только структурирует) |

### Связи между событиями

Все события логически связаны через `linkedEventIds`. Примеры цепочек:

**Цепочка 1 — шум и реакция:**
```
10:30 — food (каша с сыром)
10:32 — voice_observation (мама наговорила наблюдение)
10:35 — behavior (наблюдалась нервозность)
10:38 — communication (звук «ту-ту»)
10:40 — toilet (стул жидкий)
```

**Цепочка 2 — сенсорика → calm mode:**
```
14:20 — sensory (закрывал уши при громкой музыке)
14:25 — calm_mode (запустил Calm Mode, использовал наушники)
14:30 — state (стало спокойнее)
```

**Цепочка 3 — AAC карточка → действие:**
```
11:15 — aac_card (нажал «Хочу пить»)
11:30 — communication (звук «ва»)
```

## 6. AI insights (mock)

Mock AI-выводы формулируются на основе данных `demoDataset.ts`. Логика в `EventTimeline.tsx` и `SpecialistDashboard.tsx`:

| Контекст | AI-вывод |
|----------|---------|
| ≥2 sensory события | «Похоже, сегодня несколько событий связаны с сенсорной чувствительностью. Это наблюдение, не диагноз. Можно обсудить со специалистом.» |
| ≥2 communication/aac | «Похоже, ребёнок активно использовал коммуникацию. Это хороший знак! Нужно подтвердить наблюдениями.» |
| ≥3 food | «Похоже, сегодня было несколько событий, связанных с питанием. Это может быть полезной информацией для специалиста.» |
| Default | «Сегодня собрано достаточно наблюдений. Чем больше данных — тем точнее паттерны.» |

**Обязательные оговорки** (есть в каждом выводе):
- «Это наблюдение, не диагноз»
- «Нужно подтвердить»
- «Можно обсудить со специалистом»

## Расширение (Phase 2)

Когда понадобится больше детей или событий:

1. Добавить в `DEMO_CHILDREN` новый профиль.
2. Добавить новые события в `SEED` массив в `demoDataset.ts`.
3. Расширить `getDemoTimelineSummary` если нужны новые типы.
4. Обновить `ChildProfiles` для тьюторов и специалистов.

Все изменения делаются в одном файле — `demoDataset.ts`.
