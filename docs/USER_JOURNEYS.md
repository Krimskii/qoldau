# User Journeys — Qoldau AI

> Главные сценарии пользователей Demo MVP.
> Каждый journey — реальный путь, который можно пройти через приложение.

## Journey 1: Мама Алихана — обычный день

**Контекст:** Алия, мама Алихана (7 лет, РАС), проводит обычное утро.

### Шаги

1. **Открывает приложение** → Overview
2. **Переключается на роль Родителя** (Role Switcher)
3. **ParentHome**: видит, что Алихан «сейчас спокойный», последние события за вчера.
4. **AI-наблюдение**: «Похоже, сегодня хорошо использовать визуальное расписание».
5. **Жмёт «Сказать наблюдение»** → `/parent/voice`
6. **Записывает голосом** (mock): «Он поел кашу с сыром, потом закрывал уши, сказал „ту-ту", сходил в туалет».
7. **Останавливает запись** → `/parent/ai-review`
8. **AI показывает transcript и 3 предзаполненных события** (food, behavior, toilet).
9. **Жмёт «Подтвердить и продолжить»** → `/parent/clarify`
10. **Отвечает на 3 уточняющих вопроса**: воды — нормально, после туалета — да, был ли шум — да.
11. **Жмёт «Сохранить и подтвердить»**.
12. **Возврат в Event Timeline** — 3 новых события сверху, все с `status: confirmed`.
13. **Открывает детали первого события** → `EventDetails`.
14. **Видит связанные события** (поведение, туалет), AI-гипотезу «Похоже, нервозность связана с сенсорикой».
15. **Жмёт «В отчёт»** → toast «Событие добавлено в отчёт».
16. **Возвращается на ParentHome** → нажимает «AI-помощник».
17. **ParentAIChat**: выбирает пресет «Что помогало?» → AI отвечает на основе Event Timeline.
18. **Переключается на роль Ребёнка** → проверяет, как Алихан пользовался бы карточками.

**Что мама видит в итоге:**
- 3 новых подтверждённых события
- AI-наблюдение о возможной связи шума и нервозности
- Понимание, что «ту-ту» — подтверждённый сигнал для туалета

## Journey 2: Алихан — взаимодействие через интерфейс

**Контекст:** Алихан с помощью мамы использует приложение для коммуникации.

### Шаги

1. **Role Switcher → Ребёнок** → `/child/home`
2. **Видит 6 крупных кнопок**: Хочу пить / Туалет / Помощь / Пауза / Любимые / Сказать.
3. **Жмёт «Хочу пить»** → `/child/cards` → нажимает AAC-карточку «Вода».
4. **In-app feedback**: «✓ Я сказал: Вода. Мама получила уведомление».
5. **Создаётся Event** `type: aac_card, sourceRole: child, payload: { cardLabel: "Вода" }`.
6. **Возврат на ChildHome** → жмёт «Сказать».
7. **ChildSpeak**: держит кнопку микрофона, mock STT слышит «ва».
8. **AI предлагает**: «Похоже: вода (нужно подтвердить)».
9. **Жмёт «Да ✓»** → создаётся Event `type: communication`.
10. **Возврат на ChildHome** → жмёт «Помощь» → `/child/call`.
11. **Жмёт «Позвать маму»** → создаётся Event `type: sos`.
12. **In-app feedback**: «✓ Мама получила сигнал».
13. **Возврат на ChildHome** → жмёт «Любимые» → `/child/favorites`.
14. **Выбирает «Маша и Медведь»** → создаётся Event `type: media_request`.
15. **In-app feedback**: «✓ Мама увидит запрос».

**Что создаётся за один сценарий ребёнка:**
- 5 событий в Event Timeline: aac_card, communication, sos, media_request + 1 bonus (если «Сейчас/Потом» → state).
- Все события `sourceRole: child, status: confirmed`.
- Мама видит их в `/parent/events` без перезагрузки.

## Journey 3: Тьютор Айдана — занятие

**Контекст:** Айдана ведёт занятие с Алиханом в центре.

### Шаги

1. **Role Switcher → Тьютор** → `/tutor/home`
2. **Видит расписание**: «Сегодня: 10:00–15:00, занятие + обед».
3. **Видит подсказки** для Алихана: «Перед переходом предупреждайте за 1–2 минуты».
4. **Жмёт «Наговорить событие»** → `/tutor/voice`.
5. **Записывает голосом** (mock): «Использовал визуальное расписание, переходы прошли спокойнее, просил паузу».
6. **Останавливает** → `/tutor/ai-review`.
7. **AI показывает transcript, события, «что помогло»**.
8. **Жмёт «Сохранить в Event Timeline»** → создаётся Event `type: tutor_note`.
9. **Переход в `/tutor/report`** — видит отчёт за 7 дней:
   - 3 своих наблюдения
   - Что помогло (пауза, визуальное расписание)
   - Что уточнить дома
10. **Жмёт «Скопировать отчёт»** → toast «Отчёт скопирован в буфер обмена».
11. **Жмёт «Отправить родителю»** → toast «Отчёт отправлен родителю».
12. **Открывает «Профиль ребёнка»** → `/tutor/child-profile`.
13. **Видит сигналы Алихана**, что помогает, чего избегать, предпочтения.

**Что тьютор увидит через неделю:**
- Все tutor_note события за смену
- Паттерны (что помогало чаще)
- Отчёт для родителя

## Journey 4: Специалист Айгерим — обзор ребёнка

**Контекст:** Айгерим, специалист по коммуникации, готовится к встрече с семьёй Алихана.

### Шаги

1. **Role Switcher → Специалист** → `/specialist/dashboard`.
2. **Видит KPI за 7 дней**: 24 события, 5 новых сигналов, 3 коммуникации, +12% подтверждений.
3. **AI-summary**: «Похоже, замечена связь между шумом и нервозностью. Это наблюдение, не диагноз. Можно обсудить с семьёй».
4. **Жмёт «ABC-анализ»** → `/specialist/abc`.
5. **Видит 3 колонки A/B/C**, построенные из реальных событий:
   - A (До): «Шум (группа)», «Смена задания»
   - B (Что): «Нервозность», «Закрывает уши»
   - C (После): «Пауза», «Успокоение»
6. **Жмёт «Коммуникации»** → `/specialist/communication-profile`.
7. **Видит 5 сигналов** Алихана с confidence и источниками подтверждений.
8. **Жмёт «Паттерны»** → `/specialist/care-patterns`.
9. **Видит гипотезы**: «питание → 30 мин → сенсорная реакция».
10. **Жмёт «План поддержки»** → `/specialist/support-plan`.
11. **Видит 4 блока**: что помогает / что попробовать / что подтвердить / что обсудить.
12. **Возвращается в Dashboard** → открывает SpecialistEvents.
13. **Фильтрует по «Родитель»** → видит только голосовые наблюдения мамы.

**Что специалист выносит со встречи:**
- Полный контекст за неделю
- Подтверждённые сигналы
- ABC-паттерны
- Готовый план поддержки для обсуждения с семьёй

## Journey 5: Guided Demo Mode (для инвестора)

**Контекст:** Показ продукта инвестору на 10 минут.

### Шаги

1. **Overview** → жмёт «Запустить демо».
2. **18 шагов** с автонавигацией:
   1. Overview
   2. ParentHome — состояние ребёнка
   3. VoiceObservation — родитель говорит
   4. AIReview — AI структурирует
   5. ClarifyingQuestions — уточняем детали
   6. EventTimeline — всё в одном месте
   7. EventDetails — связанные события
   8. ChildHome — крупные кнопки
   9. ChildCards — нажатие создаёт Event
   10. ChildFavorites — выбор мультика
   11. ChildSpeak — голосовой запрос
   12. PhraseBuilder — сборка фразы
   13. TutorHome — расписание тьютора
   14. TutorAIReview — тьютор сохраняет
   15. TutorReport — отчёт с copy/send
   16. SpecialistDashboard — KPI
   17. CommunicationProfile — сигналы
   18. Возврат в Overview
3. **В конце**: «Спасибо за внимание!»

**Что инвестор видит:**
- 4 роли с реальными сценариями
- Event Timeline как ядро
- AI как помощник, не как диагност
- Полноту продукта без backend
