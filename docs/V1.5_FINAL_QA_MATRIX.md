## E7 detailed deliverables

### E7.1 — DataState on remaining screens
- **ParentAnalytics:** DataState wrapper для donut/heatmap/sensory. Empty "Пока недостаточно данных" с подсказкой.
- **BehaviorSensory:** DataState для sensoryEvents. Empty "Пока нет сенсорных событий" с CTA.

### E7.2 — tutor/specialist honest state + disclaimers + i18n
- **SpecialistDashboard:** убран broken `.replace('сенсорн','коммуникац')` в `aiSummary` (3 ветки возвращали одно и то же). Добавлен `commHint` i18n key.
- **SpecialistDashboard:** hardcoded `recentSignals` fallback помечен DemoBadge.
- **ABCAnalysis:** 3 хардкод PatternRow → i18n + DemoBadge.
- **CommunicationProfile:** Methods Used computed из реальных signals по category (был хардкод [8,6,3,2,1]). Empty state когда сигналов нет.
- **CarePatterns:** `hypothesisNote` на каждой pattern карточке + `bigPictureDisclaimer`.
- **~20 hardcoded i18n строк** → i18n keys с паритетом ru/kk/en.

### E7.3 — мультиребёнок консистентен
- **Новый хук `useCurrentChild()`** — единый источник выбранного ребёнка (selectedChildId из store, fallback DEMO_PRIMARY_CHILD).
- **ChildSelector** добавлен в: ParentHome, ParentAnalytics, BehaviorSensory, CareDiary, ParentNotifications, ParentProfile.
- **Заменены все DEMO_PRIMARY_CHILD.id** на `childId` из useCurrentChild в фильтрах событий и `startStoreRecording`.
- **VoiceObservation, ClarifyingQuestions, ParentAIChat** — childId из useCurrentChild как fallback.
- **ParentHome:** todayEvents теперь фильтруются по childId (раньше фильтра не было — все события всех детей смешивались).

### E7.4 — a11y smoke-test (12 тестов)
- aria-label: «Позвать маму», «Собрать фразу», «Назад», «Начать запись», «Очистить» — все доступны для screen reader.
- Заголовки (headings) на ChildProgress.
- Touch-targets: проверка tabIndex (все buttons достижимы).
- Keyboard nav: ни одна button не имеет tabIndex=-1.

**A11y baseline:**
- 129 aria-label в JSX
- `focus-visible:ring-2 focus-visible:ring-teal/40` на всех кнопках
- `prefers-reduced-motion: reduce` отключает анимации в `globals.css`, `animations.css`, `haptics.ts`, child icons

### E7.5 — auth-ready JWT (за VITE_REQUIRE_AUTH)
- **`api/client.ts`:** Authorization: Bearer <jwt> подмешивается автоматически через `registerAuthGetter()`.
- **401 handler:** `register401Handler()` → мягкий редирект на `/auth/login?returnTo=...` (только если `VITE_REQUIRE_AUTH=true`).
- **Demo-режим** (по умолчанию): handler зарегистрирован, но не редиректит без флага. Полностью сохранён.
- **`router.tsx`:** новый `<AppInit />` компонент внутри HashRouter регистрирует getter и handler, вызывает `useAuthStore.init()` для подтягивания сохранённой сессии из localStorage.
- **App.tsx:** убрана случайная дубликат-секция JSX.

**Когда бэкенд P1/v1.6 будет готов:** просто включить `VITE_REQUIRE_AUTH=true` — фронт уже умеет подмешивать JWT и обрабатывать 401.
