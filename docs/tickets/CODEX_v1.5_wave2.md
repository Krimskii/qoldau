# Ticket — Codex — v1.5 Wave 2 (~2ч): недельный дайджест + безопасность + евал-расширение

> Зона: **только `apps/api/**`.** STRATEGY §7.4 (дайджест), §7.5 (границы AI).

```
git switch integration/v1.5 && git pull
git switch -c feature/v1.5-backend-wave2
```
3 коммита (A→B→C). Ключи не коммитить; тесты на mock.

## Коммит A — Эндпоинт недельного дайджеста
Фронт считает недельную статистику локально (per-device); backend только
«очеловечивает» её LLM-ом. PII ребёнка НЕ хранить.
1. `POST /api/ai/digest` — принимает агрегаты (без сырых транскриптов):
   `{ childAgeBand?, counts: {behavior,communication,...}, topTriggers?, sensoryTop? }`
   → возвращает `{ text: string }` — 2–4 фразы человеческим RU, тон «Похоже…»,
   обязательно «Это наблюдение, не диагноз», без советов-предписаний.
2. Mock-fallback без ключа (шаблонный дайджест из чисел).
3. Тест контракта + инвариант «наблюдение, не диагноз».

## Коммит B — Безопасность: красные флаги (STRATEGY §7.5.4)
1. В parse/insight: если транскрипт содержит признаки самоповреждения / опасности —
   `insight` возвращает **шаблонный безопасный ответ** «Опишите это специалисту /
   обратитесь за помощью», НЕ бытовые советы; пометить `safetyFlag: true` в ответе.
2. Никаких медицинских инструкций. Тест: опасная фраза → safetyFlag + безопасный текст.

## Коммит C — Расширение евала + версия промпта
1. Добавить в `golden.ru.json` ещё 8–10 кейсов (сенсорика, ABC, красные флаги, пустой).
2. `runEval` печатает per-field breakdown (types/abc/sensory/safety).
3. Bump `promptVersion` при любой правке промпта; лог + `/api/ai/health` показывают версию.
