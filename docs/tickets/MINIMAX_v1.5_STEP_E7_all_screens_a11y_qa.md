## E7.4 — Сквозной a11y-проход (по разделу 0.4 базовой спеки)
- `aria-label` на всех кнопках без текста; `alt` на изображениях; `role`/семантика заголовков.
- `focus-visible:ring-2 ring-teal/40` на всех интерактивных; ловушка фокуса в модалках/шитах.
- Тач-таргеты: взрослый ≥44–48, детский ≥112.
- Контраст ≥ AA (проверить светлый текст на цветных фонах — типичная проблема).
- `prefers-reduced-motion` уважается во всех анимациях (pulse/slide/scale).
- Прогнать axe/линтер a11y (если настроен) или ручной чек-лист; 0 нарушений на ключевых экранах.

## E7.5 — Auth-ready (подготовка фронта к бэкенд P1/P2, БЕЗ поломки демо)
Бэкенд вводит обязательный JWT на data-роутах (P1) и sync API (P2). Подготовить фронт, но
**не сломать текущий local-first демо-режим**:
- В `api/client.ts`: добавлять `Authorization: Bearer <jwt>` из auth-стора, если токен есть.
- Обработка **401** (протух/нет токена) → мягкий редирект на `/auth/login` (не белый экран),
  **403** → `ErrorState` «нет доступа». В демо-режиме (нет токена/флаг) — старое поведение
  (local-first), без принудительного логина.
- Auth-стор: хранить jwt + user (role, доступные childId из `/api/auth/me`), logout чистит.
- LoginPage/VerifyPage: довести реальный флоу (request-magic-link → verify → сохранить jwt →
  home по роли), с состояниями/ошибками; демо-кнопка «Продолжить как демо» остаётся.
- НЕ включать обязательный логин по умолчанию (ждём, пока бэкенд P2/sync будет готов) — за
  флагом окружения `VITE_REQUIRE_AUTH`.
