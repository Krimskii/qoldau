## Контекст — что не так (диагностика)
На проде реальный LLM (gpt-4o-mini) на простом входе **«Алихан попил воды и был спокоен»**
вернул `Эмоциональная реакция / возбуждение / Коммуникация` — то есть:
- **пропустил** конкретное `water` (питьё);
- **противоречил входу**: текст «спокоен», а модель выдала «возбуждение»;
- **дженерик-заголовки** («Коммуникация») вместо конкретики.

Плюс два структурных факта:
- **eval гоняется по MOCK, а не по LLM** — `runEval` удаляет `OPENAI_API_KEY`
  ([eval/runEval.ts:93](../../apps/api/eval/runEval.ts)). Значит текущие «50%» — качество
  оффлайн-парсера; реальный LLM в CI не измеряется вовсе.
- **mock ABC цепляется только к behavior-событию** ([llmService.ts:398](../../apps/api/src/services/llmService.ts)),
  а половина ABC-кейсов golden'а (comm/food/toilet/state) behavior не создают → abc ~50%.

## Фронт 1 — Prompt `parse-ru.v4` (реальный LLM) — главный
Файл `src/services/prompts/parseRuV2.ts`. Улучшить системный промпт, **сохранив тон и
границы безопасности**. Конкретно:

1. **Точный маппинг типов** — добавить явные правила «слово → тип» + анти-примеры:
   - `попил/выпил/пьёт/вода/сок/чай/компот` → **water** (обязательно отдельным событием);
   - `поел/ел/съел/каша/суп/печенье/обед` → **food**;
   - `горшок/туалет/пописал/подгузник` → **toilet**;
   - `сказал/показал/жест/дай/просил/мама` → **communication**;
   - `спокоен/устал/сонный/возбуждён` → **state**;
   - `уснул/проснулся/спал` → **sleep**;
   - шум/свет/толпа/запах/одежда → **sensory** + `sensoryContext`.
2. **Верность входу (anti-hallucination)** — жёсткое правило: **НЕ выдумывай** эмоции/
   возбуждение/реакции, которых нет в тексте. Если сказано «спокоен/спокойный» — `state`
   ДОЛЖНО отражать спокойствие, НЕ «возбуждение». `behavior` создавай **только** при явной
   заметной реакции (плакал/кричал/убегал/бился), не на каждое наблюдение.
3. **Конкретные заголовки** — `title` = 2–4 слова, называющие само действие
   («Попил воды», «Сел на горшок»), НЕ категорию («Коммуникация», «Состояние»).
4. **Few-shot примеры** (сильнейший рычаг для gpt-4o-mini) — добавить в промпт **3–4 мини-
   примера** transcript→ожидаемый набор событий (по образцу golden-кейсов: питьё+состояние;
   сенсорика+поведение с ABC; коммуникация+еда). Держать компактно (токены).
5. **ABC** — заполнять при наличии antecedent/consequence маркеров; цеплять к наиболее
   релевантному событию (не обязательно behavior).
