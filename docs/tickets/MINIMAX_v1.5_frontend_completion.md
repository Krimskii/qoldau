## РАЗДЕЛ 0 — ГЛОБАЛЬНЫЕ ИНВАРИАНТЫ (применить к КАЖДОМУ экрану и компоненту)

Это чек-лист, который проверяется на **каждой** странице всех разделов ниже. Если экран
его не проходит — он не «готов».

### 0.1 Дизайн-токены (0 хардкода)
- Ни одного inline `#hex`, `rgb()`, произвольного `w-[..px]` цвета в JSX. Только Tailwind-токены
  (`bg-teal`, `text-ink`, `border-line`, `shadow-card`, `rounded-2xl`…) или значения из
  `tokens.ts` (`toneToColor`, `eventTypeColors`, `childFamily`, радиусы, тени).
- Градиенты/тени — только из токенов. Провести поиск `#[0-9a-fA-F]{3,6}` по `src/pages/**` и
  `src/components/**` и вычистить (кроме `tokens.ts`, `assetRegistry`, тестов).
- Радиусы/отступы/типографика — по шкале токенов (`radii`, `spacing`, `typography`).

### 0.2 Четыре состояния везде, где показываются данные
Каждый список / карточка данных / график обязан иметь:
- **loading** — через `Skeleton` (`components/ui/Skeleton.tsx`), не «прыгающий» layout;
- **empty** — через `EmptyState` (`components/ui/EmptyState.tsx`) с понятным текстом и CTA
  «как наполнить» (например «Запишите голосом → появится здесь»);
- **error** — дружелюбное сообщение + кнопка «Повторить» (не белый экран, не throw);
- **data** — основной вид.
Ни одного «пустого места» без empty-state. Ни одного бесконечного спиннера.

### 0.3 Офлайн и API
- `OfflineBanner` уже глобальный (`app/App.tsx`) — убедиться, что появляется при потере сети.
- Все сторы работают **local-first**: если `api.*` упал (сеть/CORS/4xx/5xx) — тихий fallback
  на localStorage/in-memory, БЕЗ ошибки пользователю (см. `api/client.ts`, `HealthCheckBanner`).
- Действия, меняющие данные (создать событие, удалить запись), оптимистично обновляют UI и
  переживают отсутствие бэка.

### 0.4 Доступность (a11y)
- `aria-label` на каждой кнопке без видимого текста; `alt` на изображениях.
- `focus-visible:ring-2 ring-teal/40` на всех интерактивных элементах.
- Тач-таргеты: взрослый мир ≥44–48px, детский ≥112px (см. `tokens.components`).
- Контраст текста ≥ AA. Уважать `prefers-reduced-motion` во всех анимациях.
- Роль/семантика: заголовки `h1/h2/h3`, списки, `role="dialog"` у модалок/шитов, `aria-live`
  у toast/статусов.

### 0.5 Интернационализация (ru / kk / en)
- **Ни одной хардкод-строки в JSX** — всё через `t('...')` из `i18n/locales/{ru,kk,en}.json`.
- Провести аудит: собрать все видимые строки, вынести отсутствующие ключи, **добить kk и en
  до паритета с ru** (сейчас в них по ~5 mock-меток — проверить и дополнить). Числа/даты — через
  локале-форматтеры (`utils/dateFormat`).
- Переключатель языка (если есть в профиле) реально меняет язык на всех экранах.

### 0.6 Никаких тупиков и мёртвых кнопок
- Каждая кнопка/ссылка либо навигирует, либо выполняет действие с обратной связью. Нет
  `onClick`-пустышек, нет ведущих в 404.
- На каждом не-домашнем экране — рабочая «Назад»; на модалках — «Закрыть».
- Проверить каждый маршрут из `app/router.tsx` (60+ роутов) — открывается, рендерит контент,
  навигация туда-обратно работает.
- **0 ошибок и warning в консоли** на всех экранах (React keys, controlled inputs, отсутствующие
  переводы, битые изображения).

### 0.7 Отзывчивость и платформа
- Основной кейс — мобайл (контейнер `max-w-[430px]`), но проверить планшет/десктоп-рамки
  (`layout.tabletMaxWidth/desktopMaxWidth`) — контент не растягивается уродливо.
- Safe-area (нотч/жест-бар), фиксированные элементы над BottomNav (`bottomNavClearance`).
- Capacitor: кнопка «назад» на Android, StatusBar, SplashScreen — уже в `App.tsx`, проверить.

### 0.8 Тон и дисклеймеры
- Везде, где есть аналитика/выводы: тон «Похоже…/Возможно…», обязательный дисклеймер
  «Это наблюдение, не диагноз». Никаких диагнозов/ярлыков/обещаний лечения.
