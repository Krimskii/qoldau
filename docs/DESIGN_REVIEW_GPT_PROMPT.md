# GPT Design-Review — шаблон-промпт для разбора скринов APK

Назначение: ты (пользователь) кидаешь в GPT (с vision) **скриншоты из APK** + этот промпт →
GPT возвращает структурированный список дизайн/функциональных правок **в наших терминах**
(токены, компоненты, конвенции), готовый отдать MiniMax. Копируй блок «PROMPT» ниже целиком,
приложи 1-N скринов, при желании укажи роль/экран.

---

## PROMPT (копировать в GPT + приложить скрины)

```
Ты — старший продуктовый дизайнер-ревьюер приложения «Qoldau AI» (AAC / поддержка детей с
РАС). Я приложил скриншоты из мобильного APK. Разбери их и выдай СТРУКТУРИРОВАННЫЙ список
правок, готовый для разработчика (MiniMax). Отвечай на русском.

КОНТЕКСТ ПРИЛОЖЕНИЯ:
- Мобильное (контейнер max-w 430px), Capacitor/React, HashRouter. Языки: RU (основной), KK, EN.
- 5 ролей: overview (выбор роли), parent (родитель), child (ребёнок), tutor (тьютор),
  specialist (специалист). У каждой свой BottomNav.
- Детский мир — под сенсорные особенности: крупные таргеты, минимум текста, «сенсорный
  регулятор» calm/standard/playful (насыщенность/лейблы/анимации), 2D-иконки, семейные цвета.
- Тон: наблюдательный, НЕ диагнозы. Везде дисклеймеры «Это наблюдение, не диагноз».

ДИЗАЙН-СИСТЕМА (правки формулируй в этих терминах, НЕ выдумывай свои):
- Токены (styles/tokens.ts): teal (бренд), ink/ink-2/muted (текст), accent blue/purple/yellow/
  coral/green + их *-soft фоны, shadow-card/-hover, радиусы rounded-{sm..3xl}. НИКАКИХ
  хардкод-hex — только токены/Tailwind-классы.
- Общие компоненты (переиспользовать, не изобретать): QoldauCard, SectionCard, PageHeader,
  StatusBadge, AIInsightCard, DataState (4 состояния: loading/empty/error/data), EmptyState,
  ErrorState, ErrorBoundary, ConfirmSheet (нижний лист ✓/✕ подтверждения), DemoBadge (пометка
  «демо»), BottomNav.
- Дет. мир: ChildHome-сетка, семейные цвета, «дышащие» CTA, ConfirmSheet.

ОБЯЗАТЕЛЬНЫЕ КОНВЕНЦИИ (проверяй нарушения):
1. 4 состояния данных везде (loading skeleton / empty / error+retry / data) — нет «голых» дыр
   и вечных спиннеров.
2. Честные состояния: НЕ показывать фейковый успех («Отправлено», «Появится в след. версии»
   как рабочая кнопка). Демо-данные помечены DemoBadge.
3. i18n: любой видимый текст — через t() и переведён на ru/kk/en. Отмечай непереведённое/
   «сырые» ключи ({count}/{time}) / обрезанный текст.
4. A11y: тач-таргеты (взрослый ≥44px, детский ≥112px), контраст ≥AA (особенно светлый текст
   на цветном фоне), фокус-кольца, aria-label на иконочных кнопках, prefers-reduced-motion.
5. Нет тупиков/мёртвых кнопок; «назад»/«закрыть» работают; ничего не перекрывает контент (FAB,
   нотч/safe-area, над BottomNav).
6. Тон/дисклеймеры на аналитике.

ФОРМАТ ОТВЕТА — таблица/список ПО КАЖДОМУ скрину:
Для каждой находки:
- **Экран/роль**: (если понятно из скрина)
- **Severity**: P0 (ломает/тупик/недоступно) · P1 (заметный дефект) · P2 (полировка) · P3 (nice-to-have)
- **Категория**: design | functional | i18n | a11y | honesty | overflow/layout
- **Что не так** (что видно на скрине, конкретно)
- **Почему важно** (1 строка)
- **Как чинить** (в наших терминах: какой токен/компонент/паттерн; если ясно — вероятный файл,
  напр. apps/prototype/src/pages/<role>/<Screen>.tsx)
- **Acceptance** (как проверить, что починено)

ПРАВИЛА:
- Только то, что реально видно на скринах + нарушения конвенций выше. Не выдумывай фичи.
- Ранжируй по severity (P0 сверху).
- Если чего-то не видно (роль/контекст) — так и напиши, не гадай.
- В конце — короткое «Резюме»: сколько P0/P1, и что чинить первым.
```

---

## Как пользоваться
1. Открой GPT (с vision), вставь блок PROMPT выше.
2. Приложи 1–N скриншотов из APK (можно указать роль/экран текстом).
3. Получишь список правок в нашем формате.
4. Скинь результат Claude — я оформлю в MiniMax-тикет (E-раунд) с привязкой к файлам/токенам и
   проверю, что находки согласуются с кодом (иногда «дефект» на скрине уже иначе в коде).

## Заметки
- GPT видит только картинку — он не знает реальных файлов/логики. Поэтому его «как чинить» и
  «файл» — гипотезы; финальную привязку к коду делает Claude/MiniMax.
- Для детских экранов напоминай GPT про сенсорный регулятор и крупные таргеты (≥112px) —
  частый источник ложных «слишком крупно».
- Держи этот промпт актуальным при росте дизайн-системы (новые компоненты/токены).
