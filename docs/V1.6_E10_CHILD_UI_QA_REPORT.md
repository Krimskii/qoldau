## Что сделано (E10.1 — критичная база)

### E10.1.1 — Bottom-sheet safe-area ✓
- `components/ui/ConfirmSheet.tsx`: inner div имеет `style={{ paddingBottom: 'max(env(safe-area-inset-bottom), 24px)' }}`. Кнопки ✓/✕ не прижимаются к жест-бару на Android/iOS.
- `pages/child/CallMom.tsx` (message-sheet с быстрыми фразами): то же safe-area-inset-bottom.

### E10.1.2 — bottomNavClearance ✓
- Уже было в E8.3: `AppShell.tsx` `pb-36` (144px) для child + parent main. Дополнительная проверка: PhraseBuilderPage, ChildSpeak, CalmMode, ChildWater, NeedCard — все child-экраны рендерятся через AppShell с showNav=true, поэтому pb-36 применяется автоматически.

### E10.1.3 — PhraseBuilderPage horizontal overflow ✓
- `scrollSnapType: 'x mandatory'` + `scrollSnapAlign: 'start'` на каждой карточке пресета.
- `WebkitOverflowScrolling: touch` для iOS smooth-scroll.
- `paddingRight: max(env(safe-area-inset-right), 20px)` — карточки не уходят под safe-area на устройствах с notch.

### E10.1.4 — CallMom «Срочно» + канонические типы ✓
- «Срочно» уже было с `coral-soft` + Lock + border. ConfirmSheet (coral tone) подключён. 0 хардкод-hex.
- **Бонус:** handleCall (Мама/Папа/Тьютор) теперь создаёт `'communication'` (НЕ `'sos'`). `'sos'` — только для ConfirmSheet-подтверждённого «Срочно». Это исправило тест `childCommInclusion.test.tsx` (обновлён).

### E10.1.5 — Success-честность ✓
- NeedCard: success-pop только после handleYes (после реального addEvent).
- CallMom: feedback-toast только после ConfirmSheet-confirm.
- ChildSpeak (E10.2.7): убран success-overlays, теперь простой статус.
- Нет premature success-блоков.
