# QOLDAU AI — Дизайн-исследование интерфейса (v1.5)

> Как привести UI в соответствие потребностям детей с РАС и современному дизайну —
> и при этом НЕ сделать его скучным. Автор: Claude (дизайн-направление).
> Дата: июль 2026. Связано: [STRATEGY.md](STRATEGY.md) §4, [TECH_DECISIONS.md](TECH_DECISIONS.md) Child UI Principles.

## 1. Центральный вопрос и ответ
**Вопрос:** минимал детского UI (мы только что убрали маскотов, приветствия, декор) —
не станет ли он скучным?

**Ответ (главный принцип):** для ребёнка с РАС **«интересно» ≠ «много украшений».**
Декоративная насыщенность = сенсорная перегрузка = тревога и уход из приложения.
Вовлечение приходит из трёх других источников:
1. **Отзывчивость** — нажал → мгновенная, ясная, приятная реакция (карточка
   подсветилась, произнесла слово, мягко «дышит»). Это интересно И безопасно.
2. **Персонализация** — его имя, его фото, его любимые карточки, его цвет.
3. **Настраиваемый уровень стимуляции** — РАС это спектр: одному нужна тишина и
   покой, другому — цвет и движение. Один дизайн не подойдёт всем → **регулятор**.

> Вывод: минимал был правильным шагом (убрали перегруз). Дальше добавляем не
> декор, а **взаимодействие + персонализацию + регулятор стимуляции.**
> «Спокойно по умолчанию, радостно при действии, настраиваемо под ребёнка.»

## 2. Что детям с РАС реально нужно от интерфейса (доказательная база)
- **Предсказуемость и постоянство.** Одна раскладка, элементы на своих местах —
  моторная память (критично для AAC: карточки НЕ перетасовываются).
- **Низкая когнитивная нагрузка.** Один главный экран действия, ≤6 выборов,
  ясная иерархия «одно крупное — остальное тише».
- **Конкретность > абстракции.** Понятные иконки/реальные фото > абстрактный арт;
  буквальные подписи в 1–2 слова.
- **Сенсорно-безопасная моторика.** Анимации ≤360мс, плавный easing, opacity/scale
  ≤1.05, уважать `prefers-reduced-motion`. НО осмысленный микро-фидбек — это польза,
  а не шум (подтверждает действие).
- **Ассент и контроль.** Кнопка «закрыть/домой» всегда есть, никаких принудительных
  таймеров, ребёнок понимает что произойдёт («Мама увидит запрос»).
- **Мотивация без перевозбуждения.** Мягкий предсказуемый прогресс/награда (как
  Goally token board, Tiimo), не мигающие фейерверки.

## 3. Ловушка «скучно» и как её обойти — БЕЗ перегруза
| Плохой способ «оживить» (перегруз) | Хороший способ (вовлечение) |
|---|---|
| Маскоты и анимации везде | Мягкий отклик на КАЖДОЕ действие (подсветка, звук, «дыхание») |
| Яркий пёстрый фон | Спокойный фон + один акцентный цвет ребёнка |
| Много текста и бейджей | Фото/иконка ребёнка, его имя, его карточки |
| Случайные сюрпризы | Предсказуемая награда за серию (спокойный прогресс) |
| Один «средний» дизайн для всех | **Регулятор: тихо ↔ игриво** под конкретного ребёнка |

## 4. Оценка текущего состояния
- **Детский мир:** только что минимизирован (6 карточек, чистые экраны) — хорошая
  база, но пока «плоско»: не хватает отзывчивости, персонализации, регулятора.
- **Взрослый мир (родитель/специалист):** soft-3D карточки, тёплая палитра,
  дизайн-токены (`tokens.ts`) — зрелее, но местами демо-насыщен.
- **Ассеты:** есть soft-3D PNG иконки (30 шт.), hybrid icon system — сильная база.
- **Токены:** цвет/радиусы/тени/motion уже централизованы — можно быстро вводить темы.

## 5. Дизайн-направление (система)

### 5.1 Два языка дизайна (не смешивать)
- **Детский мир** — «safe place»: крупно, спокойно, тактильно, отзывчиво,
  максимум иконок/фото, минимум текста. Дизайн подчинён ребёнку.
- **Взрослый мир** — «умный ассистент»: soft-3D, данные, инсайты, плотнее,
  информативнее. Дизайн подчинён задаче.

### 5.2 🎛 Сенсорный регулятор (ключевой механизм — новое в v1.5/v2.0)
Настройка на ребёнка (в `useChildSettingsStore`), 2–3 пресета:
- **Спокойный** — приглушённые цвета, без звука/минимум, движение почти выкл,
  высокий контраст-опция. Для сенсорно-чувствительных.
- **Стандарт** — мягкий цвет, TTS вкл, деликатные микро-анимации.
- **Игривый** — насыщеннее цвет, звук+гаптик, чуть больше движения, награды ярче.
  Для детей, ищущих стимуляцию.
Один регулятор меняет: палитру (насыщенность), motion, звук/гаптик, награды.
**Это и есть ответ «скучно/интересно» — его задаёт семья под ребёнка.**

### 5.3 Отзывчивость = вовлечение (на каждое действие)
- Тап карточки: `scale 0.96 → 1.0` (≤200мс) + мягкая подсветка + **TTS-озвучка**
  (уже есть) + опц. гаптик.
- «Готово»: короткий добрый знак (галочка/спарк), не фейерверк.
- Живые, но спокойные состояния: `breathe`/`pulse` opacity+scale ≤1.05 на главной кнопке.

### 5.4 Персонализация
- Фото/аватар ребёнка, его имя, его акцентный цвет (выбор семьи).
- «Любимые» карточки наверх, его AAC-набор.
- Прогресс/серии — про него («ты записал 5 дней подряд»).

### 5.5 Цвет, контраст, типографика
- Палитра: спокойные тёплые + `ink #071B3A`, БЕЗ медицинских зелёных/красных.
- Насыщенность управляется регулятором (тихо=приглушённо, игриво=ярче).
- Контраст-режим (WCAG AA+) как опция. Крупный, читаемый шрифт.

### 5.6 AAC-специфика (углубляем к v2.x)
- Core vocabulary, фиксированная сетка (моторная память), символ+текст, TTS.
- Карточки не перетасовываются; размер/набор — под ребёнка.

## 6. Метод: не угадывать — исследовать с семьями
Дизайн для РАС нельзя «придумать за столом» — спектр слишком широкий.
1. **Кабинетное исследование** (сделано частично): Goally, Proloquo2Go, Tiimo,
   Choiceworks, Otsimo + гайды (NPDC/autism-friendly design) → §5 отсюда.
2. **Наблюдение на пилоте** — как реальные дети реагируют: где перегруз, где скука,
   какой пресет регулятора выбирают. **Пилот = главное дизайн-исследование.**
3. **Итерация по пресетам**, а не по «одному идеальному экрану».

## 7. Фазовый план
| Фаза | Что | Когда |
|---|---|---|
| D1 | Отзывчивость: микро-фидбек + TTS + мягкие состояния на всех детских действиях | v1.5 (сейчас) |
| D2 | 🎛 Сенсорный регулятор (спокойный/стандарт/игривый) через токены-темы | v1.5→v2.0 |
| D3 | Персонализация: фото/цвет/имя/любимые ребёнка | v2.0 |
| D4 | Взрослый мир: чистка демо-насыщенности, дизайн-язык инсайтов v1.5 | v1.5 |
| D5 | AAC-углубление: core vocab, фиксированная сетка, наборы под ребёнка | v2.x |

## 8. Definition of Done (хороший дизайн v1.5)
- Детский экран спокоен по умолчанию, но **живо откликается** на каждое действие.
- Семья может настроить уровень стимуляции под своего ребёнка.
- Ноль сенсорной перегрузки, ноль medical-визуала, всё крупное и предсказуемое.
- Взрослый мир читается как «умный ассистент», не как демо.
- Направление проверено на ≥1 реальном ребёнке (пилот).
