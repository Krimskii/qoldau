# Child Interface Guide — Qoldau AI

> Правила построения детского UI. Основано на визуальных референсах.

## 1. Главные принципы

1. **Крупные зоны касания** — кнопки минимум 96×96px
2. **Минимум текста** — максимум 1–2 слова на кнопке
3. **Большие иллюстрации** — эмодзи или иконки 32–48px
4. **Pastel-фоны** — мягкие, тёплые цвета
5. **Дружелюбный персонаж** — динозавр, облако, звёзды
6. **Безопасность** — крупная SOS-кнопка
7. **Предсказуемость** — одинаковые паттерны на всех экранах

## 2. Композиция экрана

```
┌─────────────────────────────┐
│ Header (avatar / settings)   │ ← высота ~56px
├─────────────────────────────┤
│                              │
│  Hero (приветствие)           │ ← карточка с avatar и именем
│  Большие кнопки (3×2 / 4×3)  │ ← min-h-[110px]
│  Now/Next блок                │ ← таймер или список
│  BottomNav                    │
└─────────────────────────────┘
```

- Padding: 16–20px
- Gap между секциями: 16–20px

## 3. Кнопки

### Базовая кнопка ребёнка

```jsx
<button
  className="
    min-h-[110px]         {/* высота */}
    rounded-2xl           {/* скругление */}
    bg-[#E5F4FF]          {/* pastel фон */}
    border border-line     {/* мягкая граница */}
    flex flex-col
    items-center justify-center
    gap-2
    p-3
    hover:scale-[0.97]     {/* hover */}
    active:scale-[0.94]    {/* нажатие */}
    transition-transform
  "
>
  <span className="text-4xl">{emoji}</span>
  <span className="text-base font-black">{label}</span>
</button>
```

### Доступные пастельные фоны

| Цвет | Hex | Для чего |
|------|-----|----------|
| `#E5F4FF` | голубой | Вода, чистое |
| `#F2ECFF` | фиолетовый | Туалет, спокойное |
| `#FFECEC` | коралловый | SOS, стоп |
| `#FFF2D7` | жёлтый | Пауза |
| `#FFF3CE` | золотой | Любимые |
| `#E3FBF8` | бирюзовый | Сказать |
| `#FFEDEA` | розовый | Объятие |
| `#EAF8F0` | мятный | Еда |
| `#F0EBFF` | лавандовый | Коммуникация |
| `#F3F6FA` | серый | Другое |

## 5. Микрофон-кнопка (Speak)

```jsx
<button
  className="
    w-48 h-48                  {/* 192×192 */}
    rounded-full
    bg-gradient-to-br from-teal to-teal-dark
    flex items-center justify-center
  "
  style={{
    boxShadow: '0 0 0 20px rgba(0,150,136,0.08), 0 0 0 40px rgba(0,150,136,0.045), 0 24px 40px rgba(0,150,136,0.25)'
  }}
>
  <Mic className="w-24 h-24 text-white" strokeWidth={2.5} />
</button>
```

Glow-ring вокруг — фирменный элемент, привлекает внимание.

## 6. Calm Mode

- Облако-маскот с дыхательной анимацией (`animate-breathe`)
- Пастельные карточки 6 опций (музыка, дыхание, наушники, темно, пауза)
- Footer «Ты в безопасности»
- Цвет: голубой + лавандовый

## 7. Call Mom / SOS

- SOS-кнопка: 88px высота, коралловый градиент
- Контакт-карточки: аватар 64px + 2 кнопки действия
- Цвета: коралловый для SOS, бирюзовый для обычных

## 8. Now / Next

- Блок с двумя состояниями: «Сейчас» и «Потом»
- Стрелка между ними
- Таймер под ними
- Цвет: зелёный → фиолетовый

## 9. Phrase Builder

- Верх: карточка с собранной фразой (динозавр)
- Сетка слов 4 колонки
- Каждое слово — pastel-чип
- Кнопка отправки внизу (teal градиент)

## 10. ChildProgress

- Hero: 🎉 + «Сегодня получилось!» + звезда 80px
- Достижения 2×2 (большие иконки)
- Любимые карточки
- Баннер «Молодец!» (зелёный градиент)

## 11. Запрещено в child UI

- ❌ Текст больше 2 слов на кнопке
- ❌ Кнопки меньше 96×96
- ❌ Реальные медицинские термины
- ❌ «Отказ», «проблема», «не делай»
- ❌ Звуковые алерты (только toast)
- ❌ Browser alert / confirm / prompt
- ❌ Сложные формы и поля ввода

## 12. In-app feedback

После любого действия ребёнка:

1. **Toast** внизу: «Мама получила уведомление»
2. **Звуковая индикация** (опционально)
3. **Анимация нажатия** (scale)
4. **Event создаётся** в EventStore

Никаких alert / confirm / prompt.

## 13. Доступность

- Минимум 96px touch target
- Контраст текста к фону ≥ 4.5:1
- Семантические HTML
- aria-label на кнопках с иконками
- Keyboard navigation
