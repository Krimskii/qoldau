# Privacy & Consent — Pilot (v1.0)

Минимально необходимая приватность/согласие для пилота на ~20 семей.
Владелец docs: Claude. Реализацию UI-текстов делает MiniMax (зона pages/i18n).

> ⚠️ **Юридическая оговорка.** Это pilot-уровень (внутреннее тестирование, узкий
> круг семей по прямому приглашению). Перед **публичным** запуском обязателен
> обзор юриста на соответствие законодательству РК о персональных данных и
> защите детей (аналог COPPA/GDPR-Kids). Этот документ — не юридическое
> заключение, а разумный минимум для закрытого пилота.

## 1. Что снижает риск в текущей архитектуре

Per-device + stateless прокси уже минимизирует приватностные риски:
- **Данные ребёнка (события, имя, наблюдения) хранятся локально на устройстве**
  (localStorage), не в облаке, не между семьями.
- **Backend не хранит** аудио/транскрипты/события — только транзитно прогоняет
  через STT/LLM и возвращает результат. Ничего не персистится на сервере.
- **Аудио** отправляется на прокси только в момент записи, для распознавания;
  на сервере не сохраняется.

Единственный внешний поток данных: аудио → наш прокси → Whisper (OpenAI) +
Claude (Anthropic) для распознавания и структуризации. Это нужно раскрыть.

## 2. Что показать родителю (consent gate при первом запуске)

Одноразовый экран согласия ДО первой записи (реализует MiniMax):

- Коротко и простым языком:
  - «Qoldau AI — инструмент наблюдений, **не медицинское устройство**. Не
    диагностирует и не лечит.»
  - «Данные о ребёнке хранятся **на этом устройстве**. Мы не собираем их в облако.»
  - «Когда вы записываете голос, аудио отправляется на наш сервер только для
    распознавания речи и структурирования — и **не сохраняется** там.»
  - «Распознавание использует сервисы OpenAI (Whisper) и Anthropic (Claude).»
  - Чекбокс: «Я родитель/законный представитель и даю согласие.» + кнопка «Продолжить».
- Ссылка на полную политику (краткий текст ниже, §4).
- Согласие сохраняется локально (`localStorage`), экран больше не показывается.

## 3. Правила данных (реализация)

- [ ] Аудио не сохраняется на сервере (подтвердить в коде прокси — зона Codex).
- [ ] Транскрипты/события не пишутся в серверную БД (уже так — stateless).
- [ ] Никакой аналитики/трекинга по детям; Sentry — только технические ошибки,
      без PII ребёнка в payload.
- [ ] Экспорт/удаление данных — локально: «Очистить данные» стирает localStorage.
- [ ] Синтетические имена в демо-данных (уже сделано, v0.7.4).

## 4. Мини-политика (текст для in-app и раздачи семьям)

> **Qoldau AI — приватность (пилот).**
> Qoldau AI — инструмент для наблюдений за коммуникацией и состоянием ребёнка.
> Это не медицинское устройство: приложение не ставит диагнозов и не лечит.
> **Хранение.** Все данные о ребёнке (имя, события, наблюдения) хранятся только
> на вашем устройстве. Мы не храним их на серверах и не передаём третьим лицам,
> кроме случая ниже.
> **Распознавание речи.** Когда вы записываете голосовое наблюдение, аудиозапись
> отправляется на наш сервер и далее в сервисы распознавания речи (OpenAI Whisper)
> и структурирования (Anthropic Claude). Аудио и результат на наших серверах не
> сохраняются.
> **Ваш контроль.** Вы в любой момент можете удалить все данные в настройках
> приложения («Очистить данные»).
> **Дети.** Приложением управляет взрослый (родитель/представитель), который даёт
> согласие при первом запуске.

## 5. Definition of Done (приватность для v1.0)
- [ ] Consent-gate показывается до первой записи, согласие сохраняется локально.
- [ ] Мини-политика доступна из приложения.
- [ ] Подтверждено кодом: аудио/данные не персистятся на сервере.
- [ ] «Очистить данные» реально стирает локальные данные ребёнка.
- [ ] Sentry payload не содержит имени/данных ребёнка.
- [ ] (Перед публичным запуском) — обзор юриста РК.
