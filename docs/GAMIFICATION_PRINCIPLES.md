# Gamification Principles — Qoldau AI

> Мягкая геймификация для детей с РАС. Поддержка, не оценка.

## 1. Главный принцип

**Цель — поддержать коммуникацию, не оценить ребёнка.**

Мы не используем геймификацию, чтобы заставить ребёнка «стать лучше». Мы используем её, чтобы мягко отметить, что он **уже смог** использовать тот или иной навык.

## 2. Что можно

### 2.1 Подтверждение факта использования навыка

> «Ты молодец», «У тебя получается», «Спасибо, что показал»,
> «Ты смог попросить», «Я рядом».

Эти фразы **всегда истинны** — они описывают то, что уже произошло, без обещания будущего.

### 2.2 Достижения из Event Timeline

Достижения **строятся автоматически** из Event Timeline через `lib/game/achievementRules.ts`. Это значит:

- Если ребёнок попросил воду — появилось достижение «Попросил воду».
- Если собрал фразу — «Собрал фразу».
- Если использовал Calm Mode — «Сделал паузу».

Никакого ручного ввода. Никакой оценки «уровня».

### 2.3 Мягкий прогресс через `×N`

Если навык использован несколько раз — показываем `×3` рядом с галочкой. Это **счётчик**, не streak. Не «ты должен сделать ещё раз», а «уже получилось 3 раза».

### 2.4 DailyProgressStrip без давления

Горизонтальный strip показывает **только выполненные** достижения. Если ничего не выполнено — strip не рендерится (никакого «пустого» экрана).

## 3. Что нельзя

### 3.1 Streaks

> ❌ «Ты пропустил день», «Серия прервалась», «Не забывай каждый день».

Никаких streak-индикаторов. Если ребёнок сегодня не сделал ничего — это **нормально**, а не «проигрыш».

### 3.2 Рейтинги

> ❌ «Ты на 3-м месте среди 5 детей», «Уровень 7», «XP / опыт».

Мы не сравниваем детей между собой. Нет уровней, нет ranking.

### 3.3 Проигрыши

> ❌ «Плохой день», «Не получилось», «Попробуй ещё раз».

Все формулировки — нейтральные или положительные. Не выполнено = «Скоро!» (а не «Не сделал»).

### 3.4 Социальное давление

> ❌ «Мама видит, как мало ты сделал», «Тьютор заметил».

Никаких уведомлений родителю/тьютору с оценкой ребёнка. Только факты (что произошло), не интерпретации.

### 3.5 Награды и покупки

> ❌ Монеты, кристаллы, магазин за реальные действия.

Нет reward loop, нет «премиум-контента» за поведение.

### 3.6 Наказания

> ❌ Ограничение функциональности, если ребёнок «мало использовал».

Все функции доступны всегда.

## 4. Тексты — поддерживающие формулировки

| Вместо | Используем |
|--------|-----------|
| «Ты не сделал это» | «Скоро!» |
| «Попробуй ещё раз» | (ничего, или «Я рядом») |
| «Серия: 3 дня» | (ничего, или «×3» рядом с галочкой) |
| «Уровень повышен!» | (ничего — нет уровней) |
| «Плохой день» | (ничего — нет плохих дней) |

## 5. Компоненты

### `AchievementCard`
- Показывает **только факт**: иконка + название + (галочка или «Скоро!»).
- Если ребёнок использовал навык — мягкая пастельная подсветка.
- Если нет — серая обводка, `opacity-60`. **Без красного**, без «не сделал».

### `ProgressBadge`
- Маленький badge для strip.
- Используется только для **выполненных** достижений (DailyProgressStrip не показывает невыполненные).

### `DailyProgressStrip`
- Горизонтальный scrollable strip в ChildProgress.
- Если пусто — компонент ничего не рендерит (нет пустого state).

## 6. Архитектура

```ts
// Достижения — чистая функция от events:
const achievements = computeAchievements(events, childId, withinDays);

// Каждое правило — match() функция, не state:
match: (event) => event.type === 'aac_card' && event.payload?.cardLabel === 'Вода'
```

Это значит:
- Достижения **никогда не врут** — они строятся из реальных данных.
- Их можно пересчитать в любой момент (нет «забытых» достижений).
- При reset demo — всё сбрасывается автоматически.

## 7. Чек-лист перед добавлением нового достижения

- [ ] Правило соответствует **реальному событию** в Event Timeline (нет «гипотетических»).
- [ ] Текст — нейтральный или положительный («Спасибо, что попробовал», не «Ты должен»).
- [ ] Иконка из `src/components/icons/` (не эмодзи).
- [ ] Нет streak / level / rank / score.
- [ ] Не штрафует за отсутствие (невыполненные = «Скоро!», не «не сделал»).
