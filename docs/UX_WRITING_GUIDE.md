# UX Writing Guide — Qoldau AI

> Гайд по формулировкам текстов в продукте на русском языке.
> Цель: единообразный, тёплый, нейтральный тон. Никаких медицинских claims.

## Главные принципы

1. **Наблюдение, не диагноз.** Мы фиксируем факты, не ставим диагнозы.
2. **Действие, не предписание.** Мы предлагаем шаги, не назначаем лечение.
3. **Гипотеза, не утверждение.** Мы говорим «Похоже…», не «Это точно…».
4. **Подтверждение, не вера.** Мы просим подтвердить, не «поверить».
5. **Человечность, не медицинский жаргон.** Простой русский, без терминов.

## Тон голоса

### Как мы пишем

- **Тёплый, нейтральный, поддерживающий.**
- Без панибратства.
- Без формального «Вы/Ваш».
- Короткие предложения.
- Активный залог: «Ребёнок сделал», не «Было сделано ребёнком».

### Как мы НЕ пишем

- Не используем «лечение», «диагноз», «коррекция», «нарушение».
- Не даём оценок людям.
- Не обвиняем родителей или тьюторов.
- Не используем канцелярит.

## Стандартные формулировки

### Для AI-наблюдений

| Контекст | Формулировка |
|----------|--------------|
| Общая фраза | «Похоже, …» |
| Возможная причина | «Возможно, … связано с …» |
| Предупреждение | «Это наблюдение, не диагноз.» |
| Действие | «Нужно подтвердить наблюдениями.» |
| Совет | «Можно попробовать …» |
| Консультация | «Можно обсудить со специалистом.» |
| Подтверждение | «Это подтверждено N раз.» |

### Примеры правильных формулировок

✅ **Правильно:**
- «Похоже, нервозность появилась после еды.»
- «Возможно, шум связан с сенсорной реакцией.»
- «Сигнал "ту-ту" часто подтверждался как запрос туалета.»
- «Это наблюдение, не диагноз. Нужно подтвердить.»
- «Можно попробовать короткую паузу.»
- «Можно обсудить со специалистом.»
- «За неделю похоже, что шум и переходы — частые триггеры.»

❌ **Неправильно:**
- «У ребёнка сенсорная дисфункция.» (диагноз)
- «Нужно лечить аутизм.» (медицинский claim)
- «Проблемное поведение.» (стигматизация)
- «Ребёнок манипулирует.» (обвинение)
- «Нормализуем поведение.» (медицинский жаргон)
- «Поведенческое нарушение.» (классификация)
- «Патология.» (медицинский термин)
- «Ребёнок неадекватен.» (стигматизация)

### Для AI-summary на дашбордах

Всегда в формате:
```
[Наблюдение]. Это наблюдение, не диагноз. [Рекомендация-попробовать]. [Дисклеймер-обсудить].
```

Пример:
```
«Похоже, сегодня несколько событий связаны с шумом и переходами.
Это наблюдение, не диагноз. Можно попробовать короткую паузу.
Можно обсудить со специалистом.»
```

## Сценарии

### Подтверждение события

> «Событие добавлено в Event Timeline»
> «Сохранено в Event Timeline»
> «Отмечено: стало спокойнее»

### Уведомления о действиях ребёнка

> «Мама получила уведомление»
> «Мама получила сигнал»
> «Мама увидит запрос»
> «Мама увидит фразу»

### Неуверенность AI

> «Похоже: вода (нужно подтвердить)»
> «Возможно: туалет»
> «Я не уверен, что понял»

### Помощь и пауза

> «Можно попробовать тихое место»
> «Можно попробовать снизить освещение»
> «Можно предложить тактильный инструмент»
> «Можно отметить в коммуникационном профиле»
> «Можно поддержать попытку коммуникации»
> «Можно обсудить со специалистом»

## Запрещённые слова

| Слово | Замена |
|-------|--------|
| лечение | поддержка, помощь |
| диагноз | наблюдение, оценка |
| нарушение | особенность, профиль |
| патология | особенность |
| проблемное поведение | наблюдение, поведенческие паттерны |
| неадекватное поведение | поведенческие паттерны |
| манипулирует | пытается сообщить, выражает |
| исправить | поддержать |
| нормализовать | поддержать |
| рекомендация врача | шаг, который можно попробовать |

## Детский интерфейс

### Тон

- Максимально простой.
- Короткие слова.
- Без сложных терминов.
- Дружелюбный, без покровительственного тона.

### Примеры

✅ **Правильно:**
- «Хочу пить»
- «Туалет»
- «Помощь»
- «Пауза»
- «Любимые»
- «Мама»
- «Я хочу сказать»

❌ **Неправильно:**
- «Запросить жидкость»
- «Посетить санитарное помещение»
- «Привлечь внимание»

## Дисклеймер

Все экраны должны содержать disclaimer (хотя бы мелким шрифтом):

> «Это наблюдение, не диагноз. Можно обсудить со специалистом.»

Места размещения:
- Внизу ParentHome / TutorHome / SpecialistDashboard.
- В каждом AI-insight card.
- На Overview.
- В DEMO_SCRIPT.md.
- В окне «Поделиться / экспортировать».
